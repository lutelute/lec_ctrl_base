# 第12章 制御系の定常特性 - 初級編

## 1. 定常偏差入門

### 定常偏差とは何か

**定常偏差 (steady-state error)** とは、フィードバック制御系において、時間が十分に経過した後も残る目標値と出力値の差のことです。

例えば、室温を25℃に設定したエアコンで考えてみましょう。制御系が安定した後、実際の室温が24.5℃になったとします。このとき、目標値（25℃）と実際の値（24.5℃）の差0.5℃が定常偏差です。

### なぜ定常偏差が重要か

定常偏差は、制御系の性能を評価する重要な指標です。実際の応用例を見てみましょう：

- **温度制御**: エアコンやヒーターで目標温度にどれだけ正確に到達できるか
- **モーター速度制御**: 回転数を指定した値に正確に保てるか
- **ロボットアームの位置制御**: 目標位置に正確に到達できるか

定常偏差が大きいと、制御の目的が達成できません。そのため、制御系の設計では定常偏差を小さくすることが重要な課題となります。

### 定常状態と過渡状態の違い

制御系の応答は、大きく2つの状態に分けられます：

- **過渡状態 (transient state)**: 目標値が変化した直後の、応答が変動している状態
- **定常状態 (steady state)**: 十分に時間が経過し、応答が落ち着いた状態

定常偏差は、この定常状態における偏差を指します。

## 2. フィードバック制御と偏差

### フィードバック制御系の基本構成

フィードバック制御系は、以下の要素から構成されます：

```
目標値 r(t) → (+) → 偏差 e(t) → コントローラ → プラント → 出力 y(t)
              (-)  ↑                                            |
               ←────────────────────────────────────────────────┘
                           フィードバック
```

### 偏差の定義

偏差 $e(t)$ は、目標値 $r(t)$ と出力 $y(t)$ の差として定義されます：

$$e(t) = r(t) - y(t)$$

フィードバック制御では、この偏差をゼロに近づけるように動作します。

### 時間が経っても残る偏差＝定常偏差

理想的には、十分な時間が経過すれば偏差はゼロになるはずです。しかし、実際の制御系では、時間が経っても完全にゼロにならない場合があります。

定常偏差 $e_{ss}$ は、時間が無限大に近づいたときの偏差の値として定義されます：

$$e_{ss} = \lim_{t \to \infty} e(t)$$

## 3. 簡単な例題

### 単位フィードバック系の基本形

最もシンプルな単位フィードバック系を考えます。このシステムでは、伝達関数 $G(s)$ を持つプラントに対して、出力を直接フィードバックします。

### ステップ入力に対する応答

目標値として**ステップ入力** $r(t) = R \cdot u(t)$（$u(t)$ は単位ステップ関数）を与えたとき、出力がどのように変化するか観察します。

システムのゲインが $K$ の場合、Type 0 システムでは定常偏差は次のようになります：

$$e_{ss} = \frac{R}{1 + K}$$

これは、ゲイン $K$ が大きいほど定常偏差が小さくなることを示しています。

### グラフによる定常偏差の可視化

時間応答のグラフを描くと、以下のような特徴が観察できます：

- 最初は目標値と出力の差が大きい（過渡状態）
- 時間が経つにつれて差が小さくなる
- しかし、完全にはゼロにならず、一定の差が残る（定常偏差）

この定常偏差の大きさは、システムの種類（型）や入力の種類によって決まります。詳しくは中級編で学びます。

## まとめ

- **定常偏差**は、時間が十分経過した後も残る目標値と出力の差
- 制御性能を評価する重要な指標である
- フィードバック制御系では、偏差 $e(t) = r(t) - y(t)$ を小さくするように動作する
- 定常偏差の大きさは、システムの構造とゲインに依存する
